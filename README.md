# canvas-archive

A personal IaC + Python pipeline that archives Canvas LMS courses into per-course private GitHub repositories. One command per course: provision the repo with Terraform, extract assignments and supporting material via the Canvas API (with optional fetch of instructor-hosted external content), commit locally, push when approved.

## Architecture

Two halves work together:

- **Python CLI** (`canvas-archive` entry point) handles dynamic discovery: what assignments exist, what their groups are, what the descriptions contain, where embedded files live. A strategy pattern dispatches per-course extraction. The default `canvas_only` strategy works for normal Canvas usage; the `external_site` strategy handles instructors who host the actual prompts on their own server.
- **Terraform** declares the GitHub-side state: one private repository per archived course, branch defaults, bootstrap files (README, .gitignore, .gitattributes), all driven from a JSON variable file generated by the Python side.

The boundary is intentional. Dynamic discovery lives in Python, declarative infrastructure lives in HCL.

## Stack

- Python 3.12, [uv](https://github.com/astral-sh/uv) for project and dependency management
- [canvasapi](https://github.com/ucfopen/canvasapi), [markdownify](https://pypi.org/project/markdownify/), BeautifulSoup, [pypdf](https://pypi.org/project/pypdf/) for Canvas REST and content extraction
- [pyyaml](https://pyyaml.org/) for per-course profile loading
- Terraform 1.15 with the [integrations/github](https://registry.terraform.io/providers/integrations/github/latest) provider v6
- Git LFS for binary content (lab starter `.tar` files, handout `.pdf` files)
- wget for the optional external-site mirror step

## Quick start

```bash
# 1. Install dependencies
uv sync

# 2. Configure Canvas access
cp .env.example .env
# Edit .env to set CANVAS_URL and CANVAS_TOKEN
# (Canvas: Account > Settings > New Access Token)

# 3. List your Canvas courses
uv run canvas-archive list

# 4. Run the pipeline on one course
#    (provisions repo, extracts content, commits locally; no push)
uv run canvas-archive run <canvas_id>

# 5. Inspect the local archive, then push when ready
uv run canvas-archive push <canvas_id>
```

## Per-course profiles

Most courses work with the default `canvas_only` strategy: the extractor reads assignment descriptions from Canvas and writes one Markdown file per assignment, organized by Canvas Assignment Group.

For courses where the instructor distributes the actual prompts on their own website (a common pattern), drop a YAML in `profiles/`:

```yaml
canvas_id: 3506954
slug_kebab: cmpt-306-algorithms
slug_camel: CMPT306_Algorithms
strategy: external_site

external_site:
  base_url: https://example.edu/~prof/courses/CMPT306/
  assignment_patterns:
    - regex: "^lab(\\d+)$"
      candidates: ["labs/lab{n}/index.html", "labs/lab{n}/lab{n}.pdf"]
      starters: ["labs/lab{n}/lab{n}.tar"]
    - regex: "^hw(\\d+)$"
      candidates: ["homework/hw{n}/index.html", "homework/hw{n}/hw{n}.pdf"]
  handouts_dir: handouts
```

The `external_site` extractor mirrors the instructor's site, matches assignment names to URLs or PDFs via these patterns, embeds the content into the assignment `.md` (or copies the PDF verbatim with a slim pointer .md), pulls in lab starter code, and converts handout pages. The raw mirror is deleted after extraction completes.

## Properties worth knowing

- **Idempotent.** Re-runs detect no changes and skip commits. State (tfvars, Terraform, profiles) is reconciled, not replayed.
- **Safe by default.** `run` commits locally but does not push. The `--push` flag flips it. Push verification compares local HEAD against remote `main` by hash before reporting success.
- **Stale artifacts wiped.** A clean-up step removes deprecated outputs left in remote repos from earlier code versions.
- **No-fabrication scope.** Each assignment `.md` contains exactly what Canvas (or the linked external page) provides. If the instructor left the description blank, the file says so and stops.
- **PDFs preserved as PDFs.** Where the source is a PDF, the file is copied verbatim and the companion `.md` is a thin pointer. No duplicate text extraction.
- **Roll Call Attendance is filtered globally.** Canvas auto-creates that assignment when an instructor uses the attendance roll-call tool; it carries no content.

## What this repo does not include

- The archive repos themselves (one per course) are private and live separately on GitHub. This repo is the tool that provisions and populates them, not the archived data.
- Canvas access tokens, Terraform state, and per-user variable files are all gitignored.

## Why I am sharing it

A small but real demonstration of:

- Strategy plus profile pattern for handling per-instructor variation without forking the codebase
- Declarative IaC at modest scale: `for_each` over a typed map, `lifecycle.ignore_changes`, `templatefile()`, and a real provider-version migration when github provider v6 deprecated inline `default_branch` in favor of a separate `github_branch_default` resource
- Pipeline orchestration with idempotency, dry-run defaults, and post-push hash verification
- Honest scope discipline: it does the thing, it does not pretend to do more
